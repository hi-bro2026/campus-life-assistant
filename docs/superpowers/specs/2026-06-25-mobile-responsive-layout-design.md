# 全站移动端响应式排版优化设计

## 目标

在不改变桌面端页面布局和视觉风格的前提下，修复全站手机端排版错乱问题，使首页、新生指南、校园办事、关于项目、登录、注册和个人中心在 360px、375px、390px 等常见手机宽度下保持清晰、可读、可操作。

## 当前问题

- 首页 `.home-showcase-bg .hero` 的桌面端双列网格优先级高于通用移动端规则，导致手机端正文列仅约 11px 宽，标题、说明和操作区被挤成竖排。
- 顶部导航在手机端依靠自动换行，占用首屏空间较多，链接和登录操作的层级不够清楚。
- 共享悬浮工具按钮保持桌面横向宽度，部分内容位于视口之外。
- 部分页面保留过大的手机端标题、负边距、固定最小宽度或桌面间距，使页面过长或视觉层级失衡。
- `style.css` 中有媒体查询行混入字面量 `` `r`n ``，相关手机端规则可能无法正常解析。

## 方案

采用“共享移动端覆盖 + 页面专项调整”的方式：

1. 保留现有桌面端 CSS，不重写桌面布局。
2. 在共享样式表末尾建立高优先级、作用域明确的移动端覆盖层，主要断点为 `max-width: 780px`。
3. 为七个页面的共享导航加入汉堡按钮和折叠状态；桌面端继续显示原导航，手机端默认收起。
4. 对首页及各内容页面补充少量页面级移动端规则，消除固定宽度、异常网格、负边距和过大字号。
5. 复用现有颜色、圆角、阴影、背景和校园视觉素材，不进行视觉重设计。

## 导航设计

- 手机端首行显示品牌 Logo、登录或头像入口、汉堡菜单按钮。
- 汉堡按钮使用原生 `button`，包含 `aria-expanded` 和 `aria-controls`。
- 导航展开后在头部卡片内纵向排列，链接具有足够的点击高度。
- 点击导航链接、点击头部外部或按下 `Escape` 后关闭菜单。
- 页面宽度切换回桌面断点时清理移动端展开状态。
- JavaScript 不可用时，导航仍通过基础样式保持可访问，不遮挡主要内容。

## 页面布局

### 首页

- 强制 Hero 改为单列。
- Hero 内容区恢复完整宽度，标题和正文正常横排。
- 背景图作为上方视觉区域或弱化背景呈现，避免与文字争夺宽度。
- 搜索框在窄屏下纵向排列，输入框与搜索按钮占满宽度。
- 主操作按钮使用整行或均匀双列布局。
- 功能卡片在手机端使用单列，空间允许时部分短卡片使用双列。

### 新生指南

- 降低首屏标题字号和上下留白。
- 移除手机端过强的负边距与绝对定位装饰。
- 指南步骤、地图信息、清单和快速入口按单列排列。
- 长文本、表格型内容和下载入口允许换行，不产生横向滚动。

### 校园办事

- Hero、筛选区、服务列表、详情面板和联系电话卡片按单列排列。
- 筛选按钮保持两列或单列，确保文字不被压缩。
- 详情面板取消手机端粘性定位。
- 修复失效的 `max-width: 720px` 媒体查询。

### 关于项目

- 项目介绍、技术模块、路线图和贡献者卡片改为单列。
- 降低标题尺寸，移除装饰元素对正文宽度的挤压。

### 登录、注册和个人中心

- 双栏介绍与表单改为单列，表单优先显示。
- 输入框、选择框和提交按钮占满容器宽度。
- 头像、辅助插画和说明区域缩小，避免推高首屏。
- 多字段行在手机端拆成单列。

## 共用细节

- 页面左右安全边距以 14–16px 为主。
- 内容卡片圆角适当缩小，减少手机端空白浪费。
- 按钮和导航点击区域不小于约 44px。
- 图片、视频、表格和长链接设置最大宽度与安全换行。
- 悬浮助手和工具栏缩小并贴近安全区域，展开面板不超过视口宽高。
- 不通过全局 `overflow-x: hidden` 掩盖布局错误；只在确有装饰溢出的局部容器裁剪。

## 文件范围

- `css/style.css`：共享移动端覆盖与各页面移动端专项规则。
- `js/common.js`：共享汉堡菜单状态和交互。
- 七个 HTML 页面：加入统一菜单按钮及必要的可访问性属性；不改变页面正文内容。

## 验证

- 手机宽度：360×800、375×812、390×844。
- 桌面宽度：1280×720 或更宽。
- 逐页检查：
  - 无正文被挤成竖排。
  - 无非预期横向滚动。
  - 顶部菜单可展开、关闭并支持键盘。
  - 表单控件不超出卡片。
  - 卡片、图片和长文本正常换行。
  - 桌面端导航、网格、字体和视觉位置无变化。
- 运行现有语法检查，并检查浏览器控制台错误。

## 非目标

- 不重做网站配色、品牌、图片或内容。
- 不修改业务逻辑、登录数据结构、聊天接口或部署方式。
- 不为平板单独设计全新布局；平板沿用现有断点的自然过渡。
